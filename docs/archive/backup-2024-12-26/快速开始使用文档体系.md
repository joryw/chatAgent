## 🚀 立即开始

### 1. 查看文档中心

```bash
# 打开文档中心，查看所有文档索引
cat docs/README.md
```

或访问：[docs/README.md](./README.md)

### 2. 了解体系结构

```bash
# 阅读文档管理体系说明
cat docs/文档管理体系说明.md
```

或访问：[docs/文档管理体系说明.md](./文档管理体系说明.md)

### 3. 创建你的第一个文档

**步骤：**

```bash
# 1. 选择文档类型（如：功能文档）
mkdir -p docs/features/my-feature

# 2. 复制模板
cp docs/templates/feature-doc.md docs/features/my-feature/doc.md

# 3. 编辑文档
vim docs/features/my-feature/doc.md

# 4. 更新索引
vim docs/README.md
```

### 4. 迁移现有文档（可选）

如果需要整理现有文档：

```bash
# 运行迁移脚本
./migrate_docs.sh

# 查看迁移指南
cat docs/MIGRATION_GUIDE.md
```

| 分类 | 何时使用 | 示例 |

## 📝 创建文档模板

### 1. 功能文档

```bash
cp docs/templates/feature-doc.md docs/features/[feature-name]/doc.md
```

**适用于：** 详细的功能说明、API 文档

### 2. 指南文档

```bash
cp docs/templates/guide.md docs/guides/[guide-name]/guide.md
```

**适用于：** 操作教程、配置指南

### 3. 架构决策记录

```bash
cp docs/templates/adr.md docs/architecture/adr/ADR-001.md
```

**适用于：** 重要技术决策记录

### 4. 问题排查

```bash
cp docs/templates/troubleshooting.md docs/operations/troubleshooting/guide.md
```

**适用于：** 故障排查手册

## ✅ 文档规范检查清单

创建或更新文档时，确保：

- [ ] 添加了文档头部信息（时间、维护者、状态）
- [ ] 包含清晰的概述和目标读者
- [ ] 使用中文编写（专业术语可保留英文）
- [ ] 提供完整的代码示例
- [ ] 添加相关文档链接
- [ ] 更新了 docs/README.md 索引

## 💡 快速技巧

### 查找文档

```bash
# 搜索关键词
grep -r "关键词" docs/

# 使用 ripgrep（更快）
rg "关键词" docs/
```

### 批量更新

```bash
# 查找所有需要更新的文档
find docs -name "*.md" -mtime +90

# 批量添加头部信息
# （需要自定义脚本）
```

### Git 提交

```bash
# 文档变更的提交格式
git add docs/
git commit -m "docs: add new feature documentation"
```
