# DeepSeek Reasoner 推理过程展示指南

> **功能版本**: v1.1  
> **更新日期**: 2025-12-27  
> **适用模型**: DeepSeek Reasoner

## 📖 功能简介

DeepSeek Reasoner 是一个具有推理能力的AI模型，它会在生成最终答案前进行思考推理。本系统支持实时展示这个思考过程，让你能够:

- 👁️ **实时观察**: 看到模型推理的每个字符，了解其思考过程
- 🎯 **自动优化**: 思考完成后自动折叠，保持界面整洁
- 🔍 **随时查看**: 可以随时展开查看完整的思考过程

## 🚀 如何使用

### 1. 切换到 DeepSeek Reasoner 模型

#### 方式一: 使用 UI 设置面板 (推荐)

1. 点击右上角的 **⚙️ 设置图标**
2. 在 "🤖 DeepSeek 模型" 下拉菜单中选择 **deepseek-reasoner**
3. 设置会立即生效

#### 方式二: 使用命令

在聊天输入框中输入:
```
/switch deepseek
```

然后在设置中选择 reasoner 模型。

### 2. 发送需要推理的问题

推理模型特别适合以下类型的问题:

**数学推理**:
```
计算 (123 + 456) * 789 并解释步骤
```

**逻辑推理**:
```
如果所有A是B，所有B是C，那么所有A是C吗？请详细推理
```

**科学分析**:
```
解释为什么天空是蓝色的，从光学原理出发
```

**复杂问题**:
```
分析量子纠缠现象及其哲学含义
```

### 不太适合 Reasoner 的场景

- ❌ **简单问答**: "今天天气怎么样？" → 使用 deepseek-chat
- ❌ **创意生成**: "写一首诗" → 使用 deepseek-chat
- ❌ **快速响应**: 需要极快响应速度 → 使用 deepseek-chat

## 💡 使用技巧

### 1. 引导模型展示思考

在问题中明确要求展示推理过程:

```
请一步步推理：为什么 0.1 + 0.2 ≠ 0.3？
```

```
详细分析并说明你的思考过程：如何设计一个高并发系统？
```

### 2. 查看思考过程

如果思考内容已经折叠，你可以:

1. **点击折叠的区块** (显示为 "💡 思考过程")
2. 内容会展开，显示完整的推理过程
3. 再次点击可以重新折叠

### 3. 理解思考内容

思考过程通常包含:

- 🔍 **问题分析**: 模型如何理解问题
- 🧩 **知识调用**: 使用了哪些相关知识
- 🔗 **逻辑链条**: 推理的步骤和依据
- ⚖️ **权衡考虑**: 不同方案的对比
- ✅ **结论导出**: 如何得出最终答案

## 🔍 故障排查

### 问题 1: 看不到思考过程

**可能原因**: 未使用 DeepSeek Reasoner 模型

**解决方法**:
1. 检查当前模型是否为 deepseek-reasoner
2. 使用 `/config` 命令查看配置
3. 使用设置面板切换模型

### 问题 2: 思考内容没有展开

**可能原因**: UI 渲染问题

**解决方法**:
1. 刷新浏览器页面
2. 清除浏览器缓存
3. 尝试不同的浏览器

### 问题 3: 思考过程未自动折叠

**可能原因**: 网络中断或错误

**解决方法**:
1. 检查网络连接
2. 查看错误提示
3. 重新发送问题

## 🎓 示例场景

### 示例 1: 数学推理

**问题**:
```
证明：√2 是无理数
```

**思考过程** (💭 思考中...):
```
要证明√2是无理数，我将使用反证法...
假设√2是有理数，则可以表示为p/q的形式...
其中p和q是互质的整数...
[详细推理过程]
```

**正式回答** (💡 思考过程已折叠):
```
√2 确实是无理数。证明过程如下：
[简洁的最终答案]
```

### 示例 2: 逻辑分析

**问题**:
```
三个人A、B、C中，只有一个人说真话。
A说："B在说谎"
B说："C在说谎"  
C说："A和B都在说谎"
谁在说真话？
```

**思考过程** (💭 思考中...):
```
让我逐一分析每种可能...
情况1：如果A说真话...
情况2：如果B说真话...
情况3：如果C说真话...
[详细分析过程]
```

**正式回答**:
```
C在说真话。原因是...
[最终结论]
```
