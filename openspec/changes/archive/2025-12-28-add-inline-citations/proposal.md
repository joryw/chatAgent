# Change: 添加内联引用链接功能

## Why

当前系统在使用联网搜索时,虽然会在回答后显示搜索来源,但用户体验还有改进空间:

1. **引用不可点击**: 模型在回答中使用 [1]、[2] 等标记引用来源,但这些标记只是纯文本,无法点击
2. **来源查找不便**: 用户需要滚动到回答末尾的"搜索来源"部分,手动查找对应编号的链接
3. **信息断层**: 引用标记和实际链接之间存在断层,降低了信息的可追溯性

通过实现内联引用链接功能,用户将能够:
- 直接点击回答中的引用标记跳转到来源网页
- 快速验证信息的准确性和可靠性  
- 获得更流畅的阅读和研究体验

## What Changes

- **引用解析功能**: 解析模型回答中的引用标记(如 [1]、[2]、[3])
- **链接转换功能**: 将引用标记转换为可点击的 Markdown 链接
- **引用映射管理**: 维护引用编号到搜索结果 URL 的映射关系
- **流式处理支持**: 在流式生成回答时实时处理和转换引用
- **引用列表显示**: 在回答末尾添加格式化的引用列表,包含完整的来源信息

## Impact

- **受影响的 specs**: web-search (添加引用链接相关需求)
- **受影响的代码**:
  - `src/search/formatter.py` - 添加引用链接转换功能
  - `app.py` - 在流式响应中集成引用处理
  - 可能需要新增 `src/search/citation_processor.py` - 专门处理引用的工具类

## Breaking Changes

无破坏性变更。这是对现有搜索功能的增强,不影响现有功能的使用。
